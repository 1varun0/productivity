# API Specifications

Because Peak Hub utilizes Supabase as a Backend-as-a-Service, the frontend does not interact with a traditional REST API (e.g., `/api/v1/tasks`). Instead, it uses the `@supabase/supabase-js` client library, which translates JavaScript calls into PostgREST API requests.

The "API" contract is defined entirely by the database schema and Row Level Security (RLS) policies (see `DATA_MODELS.md`).

However, for operations requiring secure server-side execution, Peak Hub uses **Supabase Edge Functions**.

## Edge Function: `send-project-invite`

**Purpose**: Dispatches a secure email to a user inviting them to a workspace. The email contains a unique token generated by the database that allows them to bypass RLS and accept the invitation.

**URL Path and Method**: 
`POST https://<YOUR_SUPABASE_REF>.supabase.co/functions/v1/send-project-invite`

**Authentication Requirements**:
Requests must include a valid JWT in the Authorization header.
`Authorization: Bearer <USER_JWT>`

**Request Body Format (JSON)**:
```json
{
  "email": "colleague@example.com",
  "projectName": "Q3 Marketing Launch",
  "inviterName": "Jane Doe",
  "inviteToken": "123e4567-e89b-12d3-a456-426614174000"
}
```

**Response Format (Success)**:
```json
{
  "message": "Email sent successfully",
  "id": "resend_message_id_string"
}
```

**Error Codes / Responses**:
- `400 Bad Request`: Missing required payload parameters.
  ```json
  { "error": "Missing required fields: email, projectName, inviterName, inviteToken" }
  ```
- `401 Unauthorized`: Missing or invalid JWT in header.
- `500 Internal Server Error`: Failure to communicate with the upstream email provider (e.g., Resend).

## Common Data Query Patterns (PostgREST via JS Client)

While not traditional endpoints, these are the standard "API" interactions used throughout the codebase:

### Fetching Tasks (Optimized via RLS)
```typescript
// The API automatically filters out tasks not owned by the user via RLS
const { data, error } = await supabase
  .from('tasks')
  .select('*')
  .order('created_at', { ascending: false });
```

### Relational Fetching (Workspaces)
```typescript
// Fetches all projects the user is a member of, and includes the owner's profile data
const { data, error } = await supabase
  .from('projects')
  .select(`
    *,
    owner:owner_id(display_name, username, accent_color)
  `);
```

### Realtime Subscription (Chat API)
```typescript
// Establishes a WebSocket connection for real-time chat
const channel = supabase.channel(`project_messages:${projectId}`)
  .on(
    'postgres_changes',
    { event: 'INSERT', schema: 'public', table: 'project_messages', filter: `project_id=eq.${projectId}` },
    (payload) => {
      // payload.new contains the new chat message
    }
  )
  .subscribe();
```
